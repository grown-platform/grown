Sites

Build and publish simple websites from your workspace — a block-based editor, multiple pages, and a public link when you're ready.

Open Sites → Browse all apps

Overview

Sites is a lightweight website builder. Stack content blocks — headings, text, images, buttons, dividers and embeds — onto pages, add as many pages as you need, and preview as you go. Keep a site as a private draft while you work, then publish it to share a clean public page. No code, no separate hosting.

Features

  • Block-based editor. Compose pages from content blocks: heading, text, image, button, divider and embed.
  • Multiple pages. A site is a tree of pages, each with its own title, path and blocks — build anything from a one-pager to a small multi-page site.
  • Live preview. See pages render as you edit, so what you build is what you publish.
  • Draft & publish. Sites start private; flip a switch to publish, and your changes go live on a public view link. Unpublish any time.
  • Embeds. Drop in an embed block to surface external content via an iframe.
  • Site management. Create, rename and manage all your sites from a grid that shows draft vs published status at a glance.

How it works

Each site's page tree is serialized to JSON and stored in the platform database. The editor renders that model in the browser and saves it back as you work. Publishing flips the site's published flag and exposes it through an unauthenticated public endpoint, so a published site renders for visitors who aren't signed in.

Content model
Page tree of blocks serialized to JSON in the platform database
Publishing
Published flag + unauthenticated public view route
Blocks
Heading, text, image, button, divider, embed
Auth
Workspace single sign-on (OIDC); published sites need no account
