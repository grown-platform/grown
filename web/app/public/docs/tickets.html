Tickets

A Jira-like helpdesk for your workspace — track requests across projects, and let anyone file one through a public link, no account required.

Open Tickets → Browse all apps

Overview

Tickets organises support and request tracking into projects. Each project has its own key and numbering, configurable statuses, and an intake mode — either your team only, or a public link anyone can use without signing in. Open a project to see its queue, drill into a ticket to set status and priority, and work it through a threaded comment conversation.

Features

  • Projects. Group work into projects, each with a name, a short uppercase key (e.g. SUP) and a description. The sidebar badges every project with its open-ticket count.
  • Per-project numbering. Tickets are numbered within their project, so each gets a stable reference like SUP-12.
  • Configurable statuses. A project defines its own set of statuses; tickets move through them, and you can filter the queue by any status.
  • Priority. Every ticket carries a priority — low, normal, high or urgent — shown as a colour-coded chip.
  • Assignee. Tickets can be assigned to (and cleared from) a workspace user.
  • Comment threads. Work a ticket through a running conversation, with the requester's details, source and timestamps in view.
  • Internal notes. Mark a comment as an internal note — visible to your team but not to the requester — for triage and back-channel discussion.
  • Configurable intake. Each project is either Team only or Public link. Public projects expose a copy-to-clipboard intake URL.
  • Public submit portal. A branded, accountless form at the project's public link lets anyone file a request — name, email, subject and details — and instantly shows them their ticket reference (e.g. SUP-12) to quote later.

How it works

Authenticated members manage projects and tickets over the workspace API; status and priority changes and comments are saved as you go. Public projects additionally expose unauthenticated intake endpoints — the submit portal fetches the project's public branding by token, posts the request, and returns its KEY-N reference. A disabled or unknown link renders a graceful not-found.

Structure
Projects → tickets (KEY-N) → comments
Intake
Team only, or a public link (no account)
Workflow
Configurable statuses, priority, assignee, internal notes
