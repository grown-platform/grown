Telephony

A VoIP softphone plus Google-Voice-style texting, with a 3CX-style PBX admin console for admins — call recordings and a live activity log included.

Open Telephony → Browse all apps

Overview

Telephony turns the browser into a phone. Place and receive calls from a dialpad, text contacts from a Google-Voice-style thread view, and — if you're an admin — run the whole phone system from a PBX administration console. Calls connect peer-to-peer over WebRTC, the directory is shared with Contacts, and console actions are captured in a searchable activity log.

Softphone

  • Dialpad. A full numeric keypad to dial extensions or numbers, with a call button and backspace.
  • Call controls. Accept, mute/unmute and hang up from an in-call bar that shows the other party, the call state and a live duration timer.
  • Incoming & outgoing states. Clear states for ringing out, incoming, connecting and connected.
  • Recent calls. A history list marking calls sent, received and missed, with a one-click "call back".
  • Directory & presence. Dialable contacts come from the Contacts app; live green/grey dots show who's online.

Texting

  • Conversation threads. A Google-Voice-style two-pane view: a list of threads on the left, the open conversation on the right.
  • Send to anyone. Start a thread with a contact or type any phone number or extension.
  • Unread markers. Threads flag unread messages and clear when opened.
  • Relative timestamps. Each message carries a time, and threads show a relative "last activity" stamp.

PBX admin console Admins only

A full, 3CX-style administration console — visible only to org admins — for configuring and monitoring the phone system end to end.

  • Dashboard. Health stats for registered extensions, active calls, trunks, calls today and voicemails, plus a system status table.
  • Extensions. Manage user extensions, devices, voicemail and registration status.
  • Ring groups & call queues. Group members with ring strategies and timeouts; configure queues with agents and SLAs.
  • Auto-attendant (IVR). An editable greeting and a digit-routing map to extensions, queues or voicemail.
  • Trunks & routes. SIP trunks plus inbound (DID) and outbound (dial-pattern) routing.
  • Voicemail, music on hold & fax. Voicemail-to-email, hold music, and fax-to-email with a transmission log.
  • Phonebook. A shared phonebook for caller-ID name lookup.
  • Reports (CDR). Filterable call detail records by date, direction and status, with inline recording playback.
  • Recordings. Storage usage, a retention policy, and per-recording play, download and delete.
  • Activity log. A searchable, filterable audit trail of console actions.
  • System. Phone provisioning, SIP security and intrusion detection, backup & restore, and SMTP email settings.

How it works

Calls connect peer-to-peer over WebRTC, coordinated by a websocket signaling channel that carries presence, call invites, accept/reject/hangup and the WebRTC offer/answer and ICE exchange. The softphone needs microphone access (served over HTTPS or localhost). The admin console is gated to org admins, and its actions are recorded in the activity log.

Media
WebRTC peer-to-peer audio (STUN for NAT traversal)
Signaling
Websocket (presence, invites, offer/answer, ICE, hangup)
Backend
Asterisk-based PBX behind the admin console
Directory
Shared with Contacts (numbers become dialable extensions)
Auth
Workspace single sign-on (OIDC); admin console gated to org admins
